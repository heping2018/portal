## 用户场景与测试 *（必填）*

### 用户故事 1 - 认证方式可配置与单点登录（优先级：P1）

**描述**：系统管理员可以通过配置选择认证方式（OAuth2 Token 默认 / JWT），OAuth2 模式支持可选的单点登录（新登录挤掉旧设备），确保系统安全和灵活的会话管理。

**为何此优先级**：认证是所有功能的基础，必须在其他功能前实现。单点登录是重要的安全特性，防止账号滥用。

**独立测试**：可以通过配置切换认证方式、登录验证 token 格式、在两个设备上登录验证互踢功能来完全测试。

**验收场景**：

1. **假设**系统默认配置为 OAuth2 模式，**当**管理员登录时，**那么**系统生成 UUID 格式的 token 并存储到数据库
2. **假设**系统配置为 JWT 模式，**当**管理员登录时，**那么**系统生成 JWT 格式的 token（包含用户信息的自包含字符串）
3. **假设**OAuth2 模式下启用了 `single-device-login: true`，**当**用户在设备 B 登录时，**那么**设备 A 的 token 失效，再次请求返回 401
4. **假设**OAuth2 模式下未启用单点登录，**当**用户在多个设备登录时，**那么**所有设备的 token 都有效，可以同时使用
5. **假设**JWT 模式下，**当**用户在多个设备登录时，**那么**每个设备的 JWT 都独立有效（JWT 无状态特性）

### 用户故事 2 - 多语言企业信息访问（优先级：P1）

**描述**：B2B 客户、B2C 客户、监管机构、合作伙伴和求职者访问 Maxunitech 企业网站，可以用他们偏好的语言（中文/英文/葡萄牙语/西班牙语）获取全面的公司信息，以了解公司的业务、资质和能力。

**为何此优先级**：这是支持所有其他用户交互的基础能力。没有多语言访问，国际用户就无法与内容互动。这代表了门户网站的核心价值主张。

**独立测试**：可以通过访问首页、切换语言并验证所有静态内容（导航、公司简介、章节标题）在四种语言中正确显示，并且语言偏好能够持久保存来完全测试。

**验收场景**：

1. **假设**用户首次访问网站，**当**首页加载时，**那么**站点默认显示中文并提供所有四种语言的语言切换器
2. **假设**用户从语言切换器中选择英语，**当**页面重新加载时，**那么**所有内容以英语显示，并且语言偏好被记住供下次访问使用
3. **假设**用户之前选择了葡萄牙语，**当**他们返回站点时，**那么**站点自动显示葡萄牙语而无需重新选择
4. **假设**用户在页面之间导航，**当**他们在会话期间切换语言时，**那么**当前页面内容更新为所选语言，同时保持他们的页面位置

### 用户故事 3 - 产品发现和信息访问（优先级：P1）

**描述**：潜在客户浏览产品目录以找到符合他们需求的产品，按类别和规格过滤，查看详细的产品信息（包括参数和应用），并下载相关文档。

**独立测试**：可以通过访问产品中心、应用过滤器、查看产品详情和下载产品文档来完全测试——所有这些都不需要任何其他模块。

**验收场景**：

1. **假设**用户访问产品中心，**当**页面加载时，**那么**产品显示带有类别过滤器和搜索功能
2. **假设**用户选择类别过滤器，**当**过滤器应用时，**那么**只显示该类别中的产品
3. **假设**用户输入搜索词，**当**搜索执行时，**那么**显示符合搜索条件的产品
4. **假设**用户点击产品，**当**产品详情页加载时，**那么**所有产品规格、应用和可下载资源以所选语言显示

### 用户故事 13 - 管理员内容管理（优先级：P2）

**描述**：授权管理员登录后台系统以管理帐户和权限、配置站点导航和内容结构、创建和编辑内容（页面、新闻、荣誉、下载）、管理产品目录和证书、处理职位发布和申请、管理表单提交和线索、维护多语言内容一致性、配置 SEO 设置和管理媒体库。

**为何此优先级**：管理员后台使非技术人员能够维护内容，减少对日常更新技术团队的依赖。这对站点的长期可持续性至关重要，但次于面向公众的功能。

**独立测试**：可以通过访问管理员登录、导航管理员章节、创建/编辑内容项目、上传媒体并验证更改反映在公共站点上来完全测试。

**验收场景**：

1. **假设**管理员访问登录页面，**当**输入有效凭据时，**那么**显示管理员仪表板，并基于角色具有适当的权限
2. **假设**管理员创建新内容，**当**内容发布时，**那么**它出现在公共站点的适当章节中
3. **假设**管理员编辑产品，**当**保存更改时，**那么**更新的产品信息显示在公共产品详情页面上
4. **假设**管理员将媒体上传到媒体库，**当**上传完成时，**那么**媒体可用于内容编辑
5. **假设**收到表单提交，**当**管理员查看线索时，**那么**提交出现在线索列表中，带有时间戳和详细信息

### 边缘案例

- 当用户在翻译内容尚不可用的页面上切换语言时会发生什么？
  - 系统显示原始语言内容，并带有翻译待定的视觉指示器
- 系统如何处理过期的证书？
  - 过期证书显示"已过期"状态，并在视觉上区分，但保持可访问以供历史参考
- 当产品搜索没有结果时会发生什么？
  - 系统显示清晰的"未找到结果"消息，并提供改进搜索条件或浏览类别的建议
- 系统如何处理非常大的文件下载？
  - 系统显示下载进度并防止超时；超过大小限制的文件显示警告并提供替代访问方法
- 当用户提交带有缺失必填字段的表单时会发生什么？
  - 系统突出显示缺失字段，并显示特定的错误消息，防止提交直到所有必填字段完成
- 系统如何处理管理员同时编辑相同内容的情况？
  - 系统实现乐观锁定，如果另一个用户在内容加载后修改了内容，则发出通知
- 当媒体上传失败（文件太大、格式无效）时会发生什么？
  - 系统显示特定的错误消息，指示原因（大小限制、允许的格式）并提供更正指导
- 系统如何处理损坏或缺失的外部链接？
  - 系统记录损坏的链接以供管理员审查；用户看到友好的消息而不是错误页面
- 当站点内容超过页面显示容量时（例如，数百个产品）会发生什么？
  - 系统实现分页，默认分页大小为 20 条/页，具有可配置的页面大小和清晰的导航控件
- 系统如何处理具有不同屏幕尺寸的移动设备？
  - 响应式设计调整布局以在 PC、平板电脑和移动设备上获得最佳观看体验
- 当管理员内容编辑期间用户会话超时时会发生什么？
  - 系统在超时前发出警告并自动保存草稿内容以防止数据丢失
- 系统如何处理高峰时段的高流量？
  - 系统实施缓存和负载平衡以保持性能而不降低质量
- 当数据库连接失败时会发生什么？
  - 系统显示友好的错误页面告知用户系统暂时不可用，同时在后台实施指数退避的自动重试机制；对于读取操作，可以考虑使用 Redis 缓存的过期数据作为降级方案
- 当 Redis 缓存服务不可用时会发生什么？
  - 系统自动降级到直接查询数据库，性能降低但保持可用；对于会话管理，临时使用数据库存储；对于数据缓存，暂时禁用缓存功能

### 关键实体

- **产品**：代表公司产品，具有属性包括名称、类别、规格、应用、描述和相关的可下载资源。产品与类别和文档具有一对多关系。
- **产品类别**：代表支持多个级别的层次结构的产品分类类别。类别与父类别具有多对一关系，与产品具有一对多关系。
- **证书**：代表公司资质或认证，具有属性包括名称、颁发机构、证书编号、颁发日期、到期日期、类别和可下载文档文件。
- **证书类别**：代表按类型或监管领域分组证书的证书分类。
- **新闻文章**：代表公司新闻项目或公告，具有属性包括标题、内容、发布日期、来源、主题/标签、特色标志、置顶标志和相关媒体。
- **新闻主题**：代表用于过滤和组织相关文章的新闻分类主题。
- **职位**：代表开放职位，具有属性包括标题、地点、部门、职位描述、要求、职责、薪资范围、申请截止日期和状态（开放/关闭）。
- **求职申请**：代表提交的求职申请，具有属性包括申请人姓名、联系信息、简历/CV 文件、求职信、提交日期和相关职位。
- **解决方案**：代表业务解决方案或服务产品，具有属性包括标题、描述、目标行业、应用、益处和相关案例研究。
- **案例研究**：代表链接到解决方案的客户成功故事或实施示例。
- **内容页面**：代表一般内容页面，具有属性包括标题、内容、元信息和多语言版本。
- **导航项目**：代表站点导航菜单项目，具有属性包括标签、链接、父关系、显示顺序和可见性设置。
- **表单提交**：代表通过联系表单提交的咨询，具有属性包括提交类型、提交者信息、消息内容、提交日期和处理状态。
- **媒体文件**：代表上传的媒体资产（图片、文档、视频），具有属性包括文件名、文件类型、大小、上传日期、版本和使用元数据。
- **管理员用户**：代表后端管理员，具有属性包括用户名、电子邮件、角色、权限和活动日志。
- **角色**：代表具有定义的访问各种后端功能权限的管理员角色。
- **审计日志**：代表管理员操作的记录，用于安全和合规跟踪。

## 假设

1. **默认语言**：中文（简体）设置为网站的默认语言，反映公司的主要市场
2. **内容翻译**：初始内容将以所有四种语言提供；持续的翻译维护是内容团队的责任
3. **托管**：具有可靠正常运行时间和备份能力的标准 Web 托管基础设施
4. **管理员用户群**：初始管理员用户群估计为跨不同部门的 5-10 个用户
5. **流量**：初始流量估计为每天 1,000-5,000 次访问，成熟时扩展到 10,000+ 次；系统支持 500 并发用户
6. **内容量**：初始产品目录估计为 50-200 个产品；50+ 篇新闻存档；20+ 个证书
7. **文件存储**：媒体库存储需求估计初始为 5-10 GB，随时间增长到 50 GB+
8. **表单提交量**：估计每天 5-20 次表单提交，跨所有表单类型
9. **SEO 要求**：适用标准 SEO 最佳实践；不保证特定的搜索引擎排名
10. **浏览器支持**：网站支持现代浏览器（Chrome、Firefox、Safari、Edge）当前和以前的主要版本
11. **移动响应性**：响应式设计目标设备屏幕宽度从 320px（移动）到 2560px（大型台式机）
12. **安全性**：适用标准 Web 安全实践；所有连接都需要 SSL/TLS 加密
13. **数据保留**：表单提交和审计日志保留至少 2 年，以供合规和业务参考
14. **性能目标**：目标页面加载时间在标准宽带（10 Mbps+）上 3 秒以内
15. **内容批准**：内容发布的编辑工作流程遵循本文档中未指定的内部公司流程

8. **SSL 证书**：必须获得并安装 SSL 证书以进行安全连接
9. **分析帐户**：必须设置分析平台帐户（例如 Google Analytics）以进行跟踪集成
10. **翻译资源**：访问专业翻译服务或双语内容编辑器以进行持续的多语言维护
11. **管理员培训**：管理员用户必须接受内容管理程序的培训

## 超出范围

以下项目明确排除在此功能之外：

1. **电子商务功能**：在线购物车、支付处理或订单管理
2. **客户门户**：基于登录的客户门户，用于订单历史或帐户管理
3. **社交媒体集成**：社交媒体源、分享按钮或社交登录
4. **实时聊天**：实时聊天或客户服务聊天机器人功能
5. **个性化**：用户特定的内容推荐或个性化体验
6. **电子邮件营销**：电子邮件新闻简报订阅、管理或分发
7. **高级搜索**：具有模糊匹配、自动完成或搜索建议的全文搜索
8. **用户生成内容**：评论、评论、论坛或社区功能
9. **多媒体流**：视频托管或流媒体平台（来自第三方服务的嵌入式视频在范围内）
10. **API 开发**：用于第三方集成的 RESTful 或 GraphQL API（分析除外）
11. **移动应用程序**：本机 iOS 或 Android 移动应用程序
12. **内容迁移服务**：手动内容输入或从遗留系统迁移（提供工具，内容输入是客户的责任）
13. **持续内容创建**：除初始启动之外的定期内容编写、翻译或更新
14. **托管和基础设施管理**：服务器管理、维护或监控
15. **安全审计**：正式安全渗透测试或除标准实践之外的合规认证
